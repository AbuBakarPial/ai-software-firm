What is this setup?
Plain English explanation for beginners

🧠 Simple explanation

Imagine you hired a team of expert software engineers. But instead of real people, they're AI agents that follow very strict rules. This setup is those rules — written in files that tell the AI exactly how to behave, what to remember, what mistakes to avoid, and what quality standards to maintain. Every time you start a coding session, the AI reads these files and acts like a professional engineer, not a random chatbot.

AGENTS.md / CLAUDE.md

The "constitution" — the main rules the AI reads first. Like a company handbook. It tells the AI how to start each session, how to behave, what to never do.

MEMORY.md

The AI's notebook. After each session, it writes what it learned about YOUR project here. Next session, it reads this first. Gets smarter over time.

ERRORS.md

A list of mistakes the AI must never repeat. Pre-filled with common Flutter/web pitfalls. Every time it makes a new mistake, it adds it here.

Skills (.claude/skills/)

Specialized guides the AI reads only when needed. Like a textbook shelf — "need to do auth? read auth-patterns/SKILL.md." 45 skills total.

Hooks (.claude/hooks/)

Automatic rules that run on every action. Block dangerous commands, auto-format code, verify results. These can't be overridden by anyone.

MCP (Dart/Flutter tools)

Live connection to real Flutter/Dart docs, package info, running app inspection. Stops the AI from guessing outdated APIs.

Key insight: Most people use AI coding tools with zero configuration. You now have an AI that follows professional engineering standards automatically, every time, for any project.

3

Run onboard (reads existing code, doesn't change it)

/onboard # AI scans all existing files. Writes codebase map to MEMORY.md. # Does NOT touch existing code. Safe to run on any project.

The AI will read your existing code structure (routing, state management, folder pattern) and adapt to it — it does NOT try to rewrite your existing code to match some template.

For existing projects with bad patterns: After onboard, tell the AI: "I want to gradually migrate state management to Riverpod, but only in new code." It will follow the hybrid approach session by session.
